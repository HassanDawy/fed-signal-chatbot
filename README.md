# Fed Signal Chatbot

A retrieval-augmented system that classifies the Federal Reserve's monetary policy stance (hawkish, neutral, or dovish) from FOMC statements, with cited evidence drawn from the actual source documents. The system runs a four-way ablation across baseline, semantic, BM25, and hybrid retrieval, with an era-stratified evaluation that separates retrieval quality from the LLM's pretraining priors.

DS593 Applied LLMs · Solo Project · Spring 2026

> The full project write-up (motivation, design decisions, evaluation methodology, results, and reflection) lives in this repo as a separate document. The README only covers what you need to clone, set up, and reproduce the work.

## Brief Notes on Approach and Results

The system ingests 175 FOMC statements (2006 to present) from `federalreserve.gov`, chunks them at 256 tokens with 32-token overlap, and indexes them in two parallel stores: a ChromaDB dense index (MiniLM-L6-v2 embeddings) and a BM25 lexical index. At query time, retrieved chunks are passed to GPT-4o-mini with a chain-of-thought prompt and three few-shot examples, and the model returns a structured JSON object with stance, reasoning, and citations.

Headline finding: on date-targeted queries, BM25 alone beats hybrid (RRF) retrieval, because dense embeddings have no representation of meeting dates and pull in topically similar but wrong meetings during fusion. The post-cutoff baseline accuracy collapses to 0% (the model refuses to answer about meetings it hasn't memorized), which is the cleanest evidence that retrieval is doing real work for post-cutoff questions.

See the project write-up in this repo for the full evaluation tables, error analysis, and discussion.

## Setup

```bash
git clone https://github.com/HassanDawy/fed-signal-chatbot.git
cd fed-signal-chatbot
pip install -r requirements.txt
```

Create a `.env` file in the repo root with:

```
OPENAI_API_KEY=sk-...
FRED_API_KEY=...      # only needed if you re-run ingest.py
```

The repo already includes the scraped statements in `data/raw/`, the built ChromaDB and BM25 indexes in `data/chroma/` and `data/bm25.pkl`, and the macro indicators in `data/fred/`, so you only need an `OPENAI_API_KEY` to reproduce the evaluation or run a single query.

## How to Run

### Full pipeline (only if you want to rebuild from scratch)

```bash
python src/ingest.py        # scrape FOMC statements + pull FRED macro indicators
python src/index.py         # build ChromaDB + BM25 indexes
python src/evaluate.py      # 38 rows × 4 modes = 152 GPT-4o-mini calls
```

`ingest.py` and `index.py` are optional. Their outputs are checked in, so you can skip straight to `evaluate.py` or `generate.py`.

`evaluate.py` writes per-row CSVs to `data/eval_runs/{mode}.csv` and the era-split summary to `data/eval_summary.csv`. Useful flags:

- `--limit N` to evaluate only the first N golden rows
- `--modes baseline hybrid` to subset which modes are run
- `--k 5` to change the number of retrieved chunks (default 5)

### Example: single-query smoke test

This is the fastest way to confirm everything is wired up correctly. It runs one query against one retrieval mode and prints the parsed result:

```bash
python src/generate.py --query "Is the Fed currently hawkish?" --mode hybrid
```
You can also pass `--mode baseline`, `--mode semantic-only`, `--mode BM25-only`, and `--mode all` right after your query to get the different modes.

Expected output (truncated; some logging lines from `chromadb` and `huggingface` precede the result):

```
=== mode=hybrid ===
  stance:    neutral
  citations: ['2025-03-19', '2024-09-18', '2025-08-22']
  reasoning: (1) Phrases like 'strongly committed to supporting maximum employment
             and returning inflation to its 2 percent objective' indicate a focus
             on inflation control, but the overall tone is more about moni...
```

The model also logs the OpenAI `system_fingerprint` (e.g. `fp_ed279b101f`) so you can tell whether OpenAI changed backend state since the frozen results were generated. 

### Example: run the full ablation

```bash
python src/evaluate.py
```

This calls GPT-4o-mini 152 times (38 golden rows × 4 modes) and prints a per-mode confusion matrix plus an era-split summary table to stdout, while writing CSVs under `data/eval_runs/` and `data/eval_summary.csv`.

### EDA notebook and generated figures

`notebooks/04_paper_figures.ipynb` is a self-contained EDA pass over the eval CSVs. It does **not** re-run the evaluation — it only reads `data/eval_summary.csv` and the per-row files in `data/eval_runs/`, then produces:

- `figures/table1_ablation_summary.csv` — per-mode × per-era accuracy and hit@5
- `figures/table2_confusion_postcutoff.csv` — per-class confusion on the post-cutoff slice
- `figures/figure2_era_split.png` — baseline vs BM25 accuracy, split by era
- `figures/figure3_postcutoff_by_mode.png` — post-cutoff accuracy and hit@5 across all four modes

Open it in Jupyter or VS Code and run all cells; the `figures/` directory is created automatically.

## Reproducibility

- **Generator model:** pinned snapshot `gpt-4o-mini-2024-07-18`, `temperature=0`, JSON mode, `seed=42`. OpenAI's `system_fingerprint` is logged on every call and returned in `answer()`'s output. A fingerprint change means OpenAI altered backend state, in which case the seed no longer reproduces prior outputs and the eval should be re-run. Even with a stable fingerprint the seed is best-effort: minor prose variation is possible but stance labels and retrieval results are stable.
- **Embeddings:** `sentence-transformers/all-MiniLM-L6-v2`, deterministic at inference.
- **Chunk IDs:** `{meeting_date}_{chunk_idx:03d}` with `collection.upsert`. Re-running `index.py` is idempotent.
- **Frozen outputs:** the CSVs in `data/eval_runs/` and `data/eval_summary.csv` are the exact numbers reported in the project write-up.

## Data Access

- **FOMC statements** are scraped from the public Fed website at <https://www.federalreserve.gov/monetarypolicy/fomccalendars.htm>. The scraper uses a 1-second polite delay and a descriptive User-Agent. All 175 raw text files plus a `_manifest.csv` (URL, char count, scrape timestamp) are committed under `data/raw/`.
- **Macro indicators** (FEDFUNDS, DGS10, CPIAUCSL, UNRATE) come from the FRED API via `fredapi`. The joined CSV (one row per FOMC meeting date, as-of joined) lives at `data/fred/macro_indicators.csv` and is committed. **Note:** the FRED indicators were pulled for completeness but are not used in the current pipeline; they're available for follow-up work.
- **Golden test set** is hand-labeled and lives at `data/golden_test_set.csv` (38 meetings split into pre-cutoff and post-cutoff eras). The 3 few-shot meetings used in the prompt are deliberately excluded from this set to prevent leakage.

You do not need network access to reproduce the evaluation, only an OpenAI key.
